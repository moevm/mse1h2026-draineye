## Протокол встречи
- В 3 итерации реализованы половина сценариев:
  - Модуль авторизации и регистрации для инспектора (серверная часть - Матвеев Никита, клиентская часть - Кривошеина Дарья)
  - Модуль авторизации и регистрации для администратора (серверная часть - Матвеев Никита, клиентская часть - Дмитриева Дарья)
  - Главная страница административной панели со списком инспекций (серверная часть - Матвеев Никита, клиентская часть - Дмитриева Дарья)
  - Реализация сценария неудовлетворительного результата работы модели (Кривошеина Дарья)
  - Улучшение модели: добавление шкалы оценки, оценка степени повреждения (Бондаренко Федор)
  - Реализация тестов для нового функционала: тесты для работы модели, административной панели (Варфоломеева Арина)

  В следующей итерации планируется реализовать оставшиеся сценарии, улучшить модель:
  - Реализовать классификацию материала в модели, изменить представление результатов модели (убрать шкалу)
  - Реализация модуля управления пользователями на административной панели
  - Реализация сценария восстановления пароля
  - Реализация редактирования профиля для инспектора
  - Реализация механизма ручной синхронизации при отстутствии интернет-соединения
  - Реализация механизма обновления данных о состоянии дренажной системы у инспекторов
  
- Фото инспектора не нужно хранить на сервере.
- Механизм проверки инспекции администратор не нужен ввиду того, что мобильным приложением могут пользоваться только инспекторы.
- Представлять результат модели в виде полученного степени повреждения относительно изображения, а не в виде шкалы.
